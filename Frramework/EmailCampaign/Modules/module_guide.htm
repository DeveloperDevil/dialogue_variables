EMERGENCY TEMPLATE

In case we need to send out an emergency email campaign (i.e in case of an accident etc.) we must use the EMERGENCY Header and the EMERGENCY text variant:

  1. Use the template called EmailCampaign_1_col_BASIC_INFO when creating the campaign
  2. Go to the cmpaign's data source, and change the value for the custom variable EMERGENCY_MODE from 'N' to 'Y'
  3. Add the TEXT module, and use variant name EMERGENCY
.

BASIC TEMPLATE

The same template can be used for other information campaigns, that is not an emergency, but where we want to "strip" away Unsubscribe, soscial media etc. In that case, do not change the value for EMERGENCY_MODE in the data source, and use the normal set of content module variants.

Like / Dislike project

Idea:

Put Like and Dislike buttons under any module, enable the customer to give us feedback on the content they receive. Save data to table in responsys (JM 26/09/18)

  • In Newsletter Test folder, campaign like_dislike_jm
  • Form created is called Survey like jm
  • We use survey.js to create the form, then we style it in the html. The html is stored in campaigns - like_yes and like_no
  • Code for the form link Unfortunately I cannot print here in the html because responsys tries to run the code, look in this source code in the comment or in the HERO variant A_like <#-- ${form('like_yes', {'usedb':true})}&CAMPAIGN_NAME=${CAMPAIGN?exec}&MODULE_NAME=HERO&MODULE_HEADLINE=${MODULE.HERO_HEADLINE?exec}&MODULE_VARIANT=${MODULE.HERO_VARIANT?exec}&MODULE_BODY_TEXT=${MODULE.HERO_INTRO_TXT?exec}&WEBVERSION_EMAIL=${clickthrough('WEBVERSION')?url('ISO-8859-1')} AND ${form('like_no', {'usedb':true})}&CAMPAIGN_NAME=${CAMPAIGN?exec}&MODULE_NAME=HERO&MODULE_HEADLINE=${MODULE.HERO_HEADLINE?exec}&MODULE_VARIANT=${MODULE.HERO_VARIANT?exec}&MODULE_BODY_TEXT=${MODULE.HERO_INTRO_TXT?exec}&WEBVERSION_EMAIL=${clickthrough('WEBVERSION')?url('ISO-8859-1')}-->
    which are placed in the module code, in this case HERO variant A_like
  • The form is saving data in !FrameworkSurvey SURVEY LIKE supplemental table in two parts:
    Part 1: When a customer clicks like or dislike we save:
    RIID_
    Q_LIKE
    CAMPAIGN_NAME
    EMAIL
    MARKET
    MODULE_NAME
    MODULE_HEADLINE
    MODULE_VARIANT
    MODULE_BODY_TEXT
    WEBVERSION_EMAIL - we are able to save the webversion link in the data table.
    Part 2: When customer fills out the form
    LIKEQ1
    LIKEQ2
    LIKEQ3
    DISLIKEQ1
    DISLIKEQ2
    DISLIKEQ3
    - All questions and design of the form can be edited.
  • On form completion the customer will see a "Thankyou page" then the webversion of the original email will load.

WiFi Survey

To be launched December 2018.

  • In !FrameworkSurvey, campaign wifi_survey_real
  • Form created is called wifi_form
  • We use survey.js to create the form, then we style it in the html. The html is stored in a html file called - wifi_survey.html in /contentlibrary/!mastersurvey/wifi_survey.htm

How to make a survey?

  1. Create in survey.js
  2. Style it using previous forms styling code and call to Norwegian css file (use wifi form here)
  3. Create supplemental table to receive answers with correct columns
  4. Create form in responsys
  5. In form settings, save to results table created (number 3)
  6. Add survey.js code with html styled into a html file inside responsys
  7. Link form to use this html document

    8. Now you should have a form, html preview, results table. All you need is the link to the form.

  8. Add the form use criptic code to the forms sendData code. (function sendDataToResponsys(survey))
  9. Create your url to the form, eg could be like: dollarsign{form('wifi_form', "CAMPAIGN_NAME=" + CAMPAIGN, "G_MARKET=" + MARKET, "EMAIL_ADDRESS=" + EMAIL_ADDRESS_)}
  10. Add this url to a campaign, test the link, it should open up your form. TEST TEST TEST the form to see you have values and auto reloads after completion. Good luck!
